# OSML Network CDK Stack

This CDK stack provides shared VPC and networking infrastructure for all OSML (OversightML) components.

## Overview

The OSML-Network stack creates a VPC with public and private subnets across multiple availability zones. This shared networking infrastructure is used by all OSML child components (model-runner, tile-server, data-intake, etc.).

## Architecture

The stack provisions:

- **VPC**: Configurable CIDR range (default: 10.0.0.0/16)
- **Public Subnets**: For resources requiring internet access
- **Private Subnets**: For internal resources
- **NAT Gateways**: Configurable number for private subnet internet access
- **Internet Gateway**: For public subnet internet access

## Configuration

The stack is configured via `bin/deployment/deployment.json`, which is generated by the parent deploy script. Example configuration:

```json
{
  "account": {
    "id": "123456789012",
    "region": "us-west-2",
    "prodLike": false,
    "isAdc": false
  },
  "network": {
    "vpcCidr": "10.0.0.0/16",
    "maxAzs": 2,
    "natGateways": 1
  }
}
```

### Configuration Parameters

- `account.id`: AWS account ID
- `account.region`: AWS region for deployment
- `account.prodLike`: Whether this is a production-like environment
- `account.isAdc`: Whether this is an ADC (AWS Data Center) deployment
- `network.vpcCidr`: CIDR block for the VPC (default: 10.0.0.0/16)
- `network.maxAzs`: Maximum number of availability zones (default: 2)
- `network.natGateways`: Number of NAT gateways (default: 1)

## Outputs

The stack exports the following CloudFormation outputs for use by child components:

- `VpcId`: The VPC ID
- `VpcArn`: The VPC ARN
- `PublicSubnetIds`: Comma-separated list of public subnet IDs
- `PrivateSubnetIds`: Comma-separated list of private subnet IDs
- `AvailabilityZones`: Comma-separated list of availability zones

## Usage

### Deployment

This stack is typically deployed via the parent deploy script (`scripts/deploy.sh`), which handles configuration generation and deployment orchestration.

Manual deployment:

```bash
npm install
npm run build
npm run deploy
```

### Synthesis

To synthesize the CloudFormation template:

```bash
npm run synth
```

### Testing

Run unit tests:

```bash
npm test
```
